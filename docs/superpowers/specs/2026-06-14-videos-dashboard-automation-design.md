## 2. Technical Thesis & Architecture

Every action must be fully automated and persistent.

### A. Automatic Cloudflare R2 Uploads
1. **Compilation Output:** When the `reddit_story` pipeline finishes running FFmpeg, it produces a real MP4 at `finalPath`.
2. **Persistence:** The pipeline reads `finalPath` into memory as a buffer and calls `uploadToR2`.
3. **Storage:** Standardized filenames `${userId}-${jobId}.mp4` will be uploaded to S3-compatible Cloudflare R2.
4. **Cleanup:** Immediately delete the local temp files from the server's disk to prevent disk bloat.
5. **Database Sync:** Save the final HTTP(S) R2 URL into the `video_queue` table under the `r2_url` column.

### B. Playback and Preview Streaming
* **Unified Playing Route (`/api/video/preview/:id`):**
  * If `r2_url` starts with `http://` or `https://`, immediately issue a **302 Redirect** directly to the public R2 URL.
  * If `r2_url` is local (e.g. `local://`), fallback to streaming chunked bytes directly from the server.
  * This allows the frontend to always play any video using a simple, clean `<video src={`/api/video/preview/${id}`} />`.

## 3. Database & Route Changes

### A. New Express Routes (`server/src/routes/video.ts`)
1. **DELETE `/api/video/queue/:id`**
   * Verifies ownership.
   * If a public `r2_url` is present, extracts S3 key and calls `deleteFromR2(key)`.
   * Deletes record from `video_queue`.
2. **PATCH `/api/video/queue/:id`**
   * Verifies ownership.
   * Updates `title` or `scheduled_at` in the database.

### B. Updated Preview Route (`server/src/routes/video.ts`)
* If `r2_url` is a web address (`http://` or `https://`), send a `302 Found` redirect.

## 4. UI Changes (`client/src/pages/Videos.tsx`)

1. **State Management:**
   * Fetch active videos on mount from `GET /api/video/queue`.
   * Loading and Empty states.
2. **Rhythm / Display:**
   * Display card grid or list with video titles, creation time, channel, status badges.
3. **Playing / Preview Modal:**
   * Clicking "Play" opens a centered 9:16 vertical player loading `/api/video/preview/${id}`.
4. **Deleting:**
   * Clicking "Delete" prompts user, sends `DELETE /api/video/queue/${id}`, and updates state instantly.
5. **Editing:**
   * Clicking "Edit" displays an inline edit modal/text input to change the video's title via `PATCH /api/video/queue/${id}`.
